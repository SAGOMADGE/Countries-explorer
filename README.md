# 🌍 Countries Explorer

SPA-приложение для просмотра, поиска и сохранения стран в избранное.

Проект построен с разделением UI и источника данных, generic-хуком загрузки, глобальным состоянием через Context + `useReducer` и автоматическими UI-тестами.

**[→ Открыть живое демо](https://countries-explorer-blue.vercel.app/)**

## Возможности

- Просмотр информации о 52 странах
- Поиск по обычному и официальному названию
- Debounce поиска с задержкой 400 мс
- Фильтрация по региону
- Совместная работа поиска и фильтра
- Страница страны с языками, валютами, населением и границами
- Переход к доступным соседним странам
- Добавление и удаление стран из избранного
- Сохранение избранного между сессиями через `localStorage`
- Обработка загрузки, ошибок и пустых результатов
- Страница для неизвестного маршрута

## Ключевые технические решения

### Сервисный слой

Получение данных изолировано в `services/countries.ts`, поэтому UI-компоненты не зависят от конкретного источника.

Во время разработки изменились условия работы внешнего API, поэтому приложение было переведено на локальный JSON со снимком 52 стран. Благодаря сервисному слою страницы и компоненты не пришлось переписывать — изменилась только реализация функций получения данных.

---

### Generic-хук `useFetch<T>`

Общая логика загрузки вынесена в кастомный хук:

```ts
useFetch<T>(fetcher);
```

Хук отвечает за:

- состояние загрузки;
- хранение полученных данных;
- обработку ошибок;
- создание `AbortController`;
- передачу `AbortSignal` в сервис;
- cleanup при размонтировании или изменении `fetcher`.

Хук принимает `fetcher`-функцию вместо URL. Благодаря этому параметры можно передавать через замыкание — например, `cca3` со страницы выбранной страны.

На главной странице `fetcher` стабилизирован через `useCallback`, поэтому изменение поискового запроса или фильтра не запускает повторную загрузку списка стран.

---

### Поиск и derived state

Все страны загружаются один раз, после чего поиск выполняется локально.

Поисковый запрос проходит через `useDebounce` с задержкой 400 мс. Отфильтрованный список не хранится в отдельном `state`, а вычисляется из:

- исходного списка стран;
- поискового запроса;
- выбранного региона.

Это позволяет избежать второго источника истины и дополнительной синхронизации состояния.

---

### Context + `useReducer`

Избранное используется на нескольких маршрутах, поэтому вынесено в `FavoritesContext`.

Reducer поддерживает три действия:

```ts
ADD;
REMOVE;
CLEAR_ALL;
```

Он добавляет и удаляет страны, очищает список, защищает состояние от дублей и использует exhaustive check через `never`.

Начальное состояние читается из `localStorage` через функцию инициализации `useReducer` — один раз при создании Provider, а не при каждом рендере.

После изменений избранное сохраняется обратно в `localStorage`.

---

### Маршрутизация

Основные маршруты приложения:

```text
/                 → список стран
/favorites        → избранные страны
/country/:cca3    → страница выбранной страны
```

Код страны передаётся через динамический параметр `cca3`.

При переходе к соседней стране параметр маршрута изменяется, создаётся новый `fetcher`, после чего `useFetch` загружает соответствующие данные.

---

### Доступные и недоступные границы

Не все коды из поля `borders` присутствуют в локальном наборе данных.

Доступные коды стран заранее собраны в `Set`:

- если соседняя страна есть в наборе, её код отображается как ссылка;
- если страны нет, код отображается как некликабельный текст.

Это предотвращает переходы на страницы без данных.

## Тестирование

Для автоматических тестов используются:

- Vitest
- React Testing Library
- `userEvent`
- `jest-dom`
- `MemoryRouter`
- `createMemoryRouter`
- fake timers
- mocks сервисного слоя

Тесты проверяют наблюдаемое поведение интерфейса, изменения DOM, работу маршрутизации и синхронизацию с `localStorage`.

### Покрытые сценарии

- `normalize`: подготовка строк для поиска
- reducer: `ADD`, `REMOVE`, `CLEAR_ALL`, защита от дублей
- `useDebounce`: начальное значение, задержка и обновление
- `CountryCard`: данные страны, флаг, ссылка и состояние кнопки
- `FavoritesPage`: empty state, восстановление из `localStorage`, удаление и сохранение результата
- `HomePage`: loading, error, success, debounce, поиск, фильтрация и пустой результат
- `DetailsPage`: загрузка по `cca3`, ошибки, доступные и недоступные границы
- переход между динамическими маршрутами соседних стран
- отсутствие повторной загрузки списка при изменении поиска

## Код-ревью и доработки

Проект прошёл код-ревью у практикующего senior-разработчика.

По итогам были внесены следующие изменения:

- внешний источник данных заменён локальным JSON без изменения UI-компонентов;
- недоступные соседние страны перестали отображаться как рабочие ссылки;
- доступные коды стран вынесены в `Set`;
- `fetcher` стабилизирован через `useCallback`;
- добавлена защита reducer от повторного добавления страны;
- расширены тесты async UI, debounce, избранного и маршрутизации;
- добавлены проверки изменений DOM после пользовательских действий;
- добавлена проверка синхронизации избранного с `localStorage`;
- добавлены тесты перехода между динамическими маршрутами.
